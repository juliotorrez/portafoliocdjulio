Historia y Origen de Python
De un proyecto navideño a dominar la IA y la ciencia de datos

Python fue creado por el programador neerlandés Guido van Rossum a finales de los años 1980. Durante las vacaciones de Navidad de 1989, mientras trabajaba en el Centro de Matemáticas e Informática (CWI) de Ámsterdam, Guido comenzó a escribir el intérprete de un nuevo lenguaje que sería sucesor del lenguaje ABC. Su nombre no viene de la serpiente, sino del programa de comedia británico Monty Python's Flying Circus, del que Guido era fanático.

La primera versión pública, Python 0.9.0, se lanzó en febrero de 1991. Desde entonces el lenguaje ha evolucionado para convertirse en uno de los más usados del mundo, siendo el lenguaje número uno en inteligencia artificial, ciencia de datos, automatización y enseñanza de programación.

1989
Creación del primer intérprete
Guido van Rossum comienza a escribir Python como proyecto navideño personal en Amsterdam.
1991
Python 0.9.0 — Primera versión pública
Se publica con funciones básicas: clases, herencia, manejo de excepciones y tipos centrales.
1994
Python 1.0
Incluye herramientas funcionales: lambda, map, filter y reduce.
2000
Python 2.0 — Comprensión de listas y recolección de basura
Agrega list comprehensions, recolección de basura cíclica y soporte Unicode.
2008
Python 3.0 — La gran reescritura
Versión que no es retrocompatible con Python 2. Limpia inconsistencias y mejora Unicode, I/O y más.
2020
Fin de soporte de Python 2
Python 2 llega a su fin de vida. Todo el mundo migra a Python 3. El ecosistema de IA explota.
Hoy
Lenguaje #1 del mundo
Python lidera los índices TIOBE y Stack Overflow. Es el lenguaje estándar de IA, ML y data science.
02
Características Principales
¿Por qué Python es tan popular y poderoso?
📖
Sintaxis Legible
Python usa indentación obligatoria en lugar de llaves. El código se parece al pseudocódigo en inglés, haciéndolo muy fácil de leer y aprender.
🔄
Interpretado
No necesita compilación previa. El intérprete ejecuta el código línea por línea, lo que facilita el desarrollo y la depuración.
🌐
Multiplataforma
El mismo código Python funciona en Windows, macOS y Linux sin modificaciones. Un código, muchas plataformas.
🦆
Tipado Dinámico
No necesitas declarar el tipo de una variable. Python lo infiere automáticamente en tiempo de ejecución. Esto se llama "duck typing".
📦
Multiparadigma
Soporta programación imperativa, orientada a objetos y funcional. Elige el paradigma que mejor se adapte a tu problema.
🏗️
Baterías incluidas
Su librería estándar es enorme: manejo de archivos, redes, matemáticas, fechas, JSON, expresiones regulares y mucho más, sin instalar nada extra.
🐍
El Zen de Python: Python tiene una filosofía de diseño plasmada en el "Zen of Python" (PEP 20). Puedes verla ejecutando import this en cualquier intérprete Python. Principios clave: "Bello es mejor que feo", "Explícito es mejor que implícito", "Simple es mejor que complejo".

Variables y Tipos de Datos
Cómo almacenar y clasificar la información en Python

En Python, una variable es un nombre que apunta a un valor almacenado en memoria. No es necesario declarar el tipo: Python lo detecta automáticamente. Las variables se crean con el operador de asignación =. Los nombres deben comenzar con letra o guion bajo, no con número, y son sensibles a mayúsculas.

Funciones
Reutilización de código: definición, parámetros, retorno y lambdas

Las funciones son bloques de código reutilizables que realizan una tarea específica. Se definen con la palabra clave def y se invocan con su nombre seguido de paréntesis. Son fundamentales para organizar programas grandes y evitar la repetición de código (principio DRY: Don't Repeat Yourself).

funciones.py FUNCIONES 
# ===== FUNCIÓN BÁSICA =====
def saludar(nombre):
    """Docstring: describe qué hace la función."""
    return f"¡Hola, {nombre}!"

print(saludar("Cultura Digital"))  # → ¡Hola, Cultura Digital!

# ===== PARÁMETROS CON VALOR POR DEFECTO =====
def potencia(base, exponente=2):
    return base ** exponente

print(potencia(5))       # → 25 (usa exponente=2)
print(potencia(2, 10))   # → 1024

# ===== *args y **kwargs: cantidad variable de argumentos =====
def sumar(*numeros):            # *args → tupla
    return sum(numeros)

print(sumar(1, 2, 3, 4, 5))   # → 15

def info_persona(**datos):      # **kwargs → dict
    for k, v in datos.items():
        print(f"  {k}: {v}")

info_persona(nombre="Ana", edad=17, semestre=2)

Programación Orientada a Objetos
Clases, objetos, herencia, encapsulamiento y polimorfismo

La Programación Orientada a Objetos (POO) es un paradigma que organiza el código en objetos, que combinan datos (atributos) y comportamiento (métodos). Python es un lenguaje orientado a objetos de pura cepa: incluso los tipos básicos como int y str son objetos. Los cuatro pilares de la POO son: Encapsulamiento, Herencia, Polimorfismo y Abstracción.

poo.py POO 
# ===== CLASE BASE =====
class Persona:
    """Clase que representa a una persona."""

    # Atributo de clase (compartido por todos)
    especie = "Homo sapiens"

    def __init__(self, nombre, edad):
        # Atributos de instancia
        self.nombre = nombre     # público
        self.edad   = edad
        self._email = None      # _  = convención "privado"

    def presentarse(self):
        return f"Hola, soy {self.nombre} y tengo {self.edad} años."

    def __repr__(self):         # representación del objeto
        return f"Persona('{self.nombre}', {self.edad})"


# ===== HERENCIA =====
class Estudiante(Persona):     # hereda de Persona
    def __init__(self, nombre, edad, matricula):
        super().__init__(nombre, edad)  # llama al padre
        self.matricula = matricula
        self.materias  = []

    def inscribir(self, materia):
        self.materias.append(materia)

    def presentarse(self):     # polimorfismo: sobreescribe el método
        base = super().presentarse()
        return f"{base} Matrícula: {self.matricula}"


# ===== USO =====
ana = Estudiante("Ana", 17, "2024-001")
ana.inscribir("Cultura Digital")
ana.inscribir("Python")
print(ana.presentarse())
print(ana.materias)
print(isinstance(ana, Persona))    # → True
print(isinstance(ana, Estudiante))  # → True

print(rd.randint(1, 100))           # número aleatorio
print(datetime.now())               # fecha y hora actual

# ===== MANEJO DE EXCEPCIONES =====
def dividir(a, b):
    try:
        resultado = a / b
    except ZeroDivisionError:
        print("Error: no se puede dividir entre 0")
        return None
    except TypeError as e:
        print(f"Error de tipo: {e}")
        return None
    else:
        print("División exitosa")    # si NO hay excepción
        return resultado
    finally:
        print("Esto siempre se ejecuta")  # siempre
