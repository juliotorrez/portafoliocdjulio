¿Qué es Arduino?
Historia, filosofía y ecosistema de la plataforma

Arduino es una plataforma de electrónica de código abierto (open-source) que combina hardware y software accesibles para estudiantes, artistas, diseñadores, hobbistas y cualquier persona interesada en crear objetos o entornos interactivos. Fue creada en 2005 en el Instituto de Diseño de Interacción de Ivrea, Italia, por Massimo Banzi, David Cuartielles, Tom Igoe, Gianluca Martino y David Mellis.

Su nombre viene de un bar de Ivrea llamado "Bar di Re Arduino" donde el equipo se reunía. El objetivo era crear una herramienta barata, fácil de usar y con comunidad activa para que cualquier persona pudiera hacer proyectos de electrónica sin necesitar un título de ingeniería.

Arduino consta de dos partes fundamentales: una placa de hardware (el microcontrolador y sus componentes de soporte) y el IDE de Arduino (entorno de desarrollo integrado), donde se escribe el código en un lenguaje basado en C/C++ que luego se carga en la placa. Hoy existen más de 30 modelos oficiales de placas Arduino.

🌐
Open Source Hardware: Los esquemas de circuito y el código fuente de Arduino son completamente públicos. Cualquier persona puede fabricar su propia placa compatible. Esto ha generado un ecosistema masivo de clones y placas derivadas como el ESP8266, ESP32, STM32 que utilizan el mismo IDE.

estructura_basica.ino ESTRUCTURA ARDUINO 
// ===== Estructura de todo programa Arduino =====
// Un programa se llama "sketch" y tiene dos funciones obligatorias:

#define LED_PIN 13    // constante: pin del LED integrado

// setup() — se ejecuta UNA SOLA VEZ al encender o resetear
void setup() {
  pinMode(LED_PIN, OUTPUT);   // configura pin 13 como salida
  Serial.begin(9600);          // abre puerto serie a 9600 baudios
  Serial.println("Arduino listo!");
}

// loop() — se ejecuta en BUCLE INFINITO mientras haya energía
void loop() {
  digitalWrite(LED_PIN, HIGH); // enciende LED
  delay(1000);                  // espera 1000ms = 1 segundo
  digitalWrite(LED_PIN, LOW);  // apaga LED
  delay(1000);
}

El Microcontrolador
El cerebro de Arduino: ATmega328P y sus características internas
🧠
ATmega328P — Microcontrolador Principal
🔧 Unidad Central de Procesamiento

El ATmega328P de Atmel (ahora Microchip) es el corazón del Arduino Uno. Es un microcontrolador de 8 bits basado en la arquitectura AVR RISC. Todo lo que hace Arduino —leer sensores, controlar motores, comunicarse— pasa por este chip. Integra en un solo encapsulado la CPU, memoria, puertos de entrada/salida, conversores analógico-digitales y módulos de comunicación.

💾
Los 3 tipos de memoria del ATmega328P:
Flash (32 KB) — Donde se almacena el sketch (programa). Se borra con cada carga.
SRAM (2 KB) — Memoria de trabajo: variables, pila de ejecución. Se borra al apagar.
EEPROM (1 KB) — Memoria permanente: guarda datos aunque se quite la energía. Ciclos de escritura limitados (~100,000).
✅ Ventajas
  • Bajo consumo energético, ideal para proyectos con batería
  • Muy bien documentado y con enorme comunidad
  • Múltiples modos de bajo consumo (sleep modes)
  • Integra ADC de 10 bits de alta precisión
  • Precio muy bajo (~$2-5 USD el chip solo)
  • Reloj de tiempo real interno calibrado
⚠️ Limitaciones
  • Solo 2 KB de RAM: muy limitado para datos grandes
  • No tiene Wi-Fi ni Bluetooth integrado
  • 8 bits: no ideal para cálculos en coma flotante
  • 32 KB de Flash: programas complejos pueden no caber
  • Sin sistema operativo: solo un programa a la vez

Pines Digitales (D0 – D13)
📶 Entradas y Salidas Digitales

Los pines digitales solo reconocen dos estados: HIGH (5V / 1 lógico) y LOW (0V / 0 lógico). Pueden configurarse como entrada (leer botones, sensores digitales) o como salida (encender LEDs, activar relés). Los pines marcados con ~ también soportan PWM.

Resistencia pull-up

✅ Ventajas
  • Fáciles de usar con digitalWrite() y digitalRead()
  • Resistencia pull-up interna activable por software
  • Compatibles con la mayoría de módulos del mercado
  • Pines de interrupción (D2, D3) reaccionan sin polling
⚠️ Limitaciones
  • Máximo 40 mA por pin: no pueden manejar motores directamente
  • Solo HIGH o LOW: sin valores intermedios sin PWM
  • Voltaje 5V: puede dañar sensores de 3.3V sin divisor
pines_digitales.ino DIGITAL 
const int LED   = 13;
const int BOTON = 2;

void setup() {
  pinMode(LED,   OUTPUT);
  pinMode(BOTON, INPUT_PULLUP); // pull-up interno activado
}

void loop() {
  int estado = digitalRead(BOTON); // leer botón

  if (estado == LOW) {  // LOW = presionado (pull-up)
    digitalWrite(LED, HIGH); // encender LED
  } else {
    digitalWrite(LED, LOW);  // apagar LED
  }
}

// PWM: simular voltaje analógico con señal digital
void ejemploPWM() {
  analogWrite(9, 0);    // 0% brillo   (0V equivalente)
  analogWrite(9, 127);  // 50% brillo  (2.5V equivalente)
  analogWrite(9, 255);  // 100% brillo (5V equivalente)
}

Pines Analógicos (A0 – A5)
📈 Conversores Analógico-Digital (ADC)

Los pines analógicos A0-A5 pueden leer voltajes continuos de 0V a 5V y convertirlos a un número entero de 0 a 1023 (resolución de 10 bits). Son esenciales para sensores que producen señales continuas: temperatura, luz, humedad, presión, posición de potenciómetros, etc.

Uso como digital
Sí (A0=D14...)
pines_analogicos.ino ANALÓGICO 
void setup() {
  Serial.begin(9600);
}

void loop() {
  int valor = analogRead(A0);   // leer pin A0 (0-1023)

  // Convertir a voltaje real
  float voltaje = valor * (5.0 / 1023.0);

  // Mapa de valores: de 0-1023 a 0-255
  int brillo = map(valor, 0, 1023, 0, 255);
  analogWrite(9, brillo);         // controlar LED con potenciómetro

  Serial.print("Valor: "); Serial.print(valor);
  Serial.print("  Voltaje: "); Serial.println(voltaje);
  delay(100);
}

🔋
Sistema de Alimentación de Arduino
⚡ Gestión de Energía

Arduino puede recibir energía de tres fuentes: el puerto USB (desde la computadora, 5V / 500 mA), el jack de alimentación externa (7–12V DC recomendado, acepta hasta 20V con pérdida de calor), o directamente por el pin Vin. Un regulador de voltaje integrado (NCP1117) convierte la entrada a 5V estables. También incluye un regulador a 3.3V (150 mA máx).

✅ Ventajas
  • Selección automática de fuente (USB vs. externa)
  • Protección contra polaridad inversa en versiones recientes
  • Salida de 3.3V integrada para sensores modernos
  • Puede alimentarse con baterías 9V de forma sencilla

  • No hay protección automática contra sobrecorriente en pines

🌡️
DHT22 — Sensor de Temperatura y Humedad
🌿 Sensor Ambiental Digital

El DHT22 (también llamado AM2302) mide temperatura ambiente (−40°C a +80°C) y humedad relativa (0% a 100%) con alta precisión. Usa un protocolo de comunicación digital de un solo cable. Es la versión mejorada del popular DHT11, con mayor rango y mejor resolución.

0% – 100% RH

±2% RH
Voltaje
3.3V – 6V
Muestreo
Cada 2 segundos
✅ Ventajas
  • Un solo pin de datos, muy fácil de conectar
  • Salida digital calibrada de fábrica
  • Librería Arduino de uso simple (DHT.h)
  • Mayor precisión que el DHT11
⚠️ Limitaciones
  • Solo una lectura cada 2 segundos
  • Sensible a la condensación extrema
  • Requiere resistencia de pull-up de 10 kΩ
dht22.inoSENSOR
#include <DHT.h>
#define DHTPIN  4       // pin de datos
#define DHTTYPE DHT22  // modelo del sensor
DHT dht(DHTPIN, DHTTYPE);

void setup() {
  Serial.begin(9600);
  dht.begin();
}
void loop() {
  float temp = dht.readTemperature();
  float hum  = dht.readHumidity();
  Serial.print("Temp: "); Serial.print(temp); Serial.print("°C  ");
  Serial.print("Humedad: "); Serial.print(hum); Serial.println("%");
  delay(2000);
}

HC-SR04 — Sensor Ultrasónico de Distancia
📏 Sensor de Proximidad

El HC-SR04 utiliza ultrasonido (40 kHz) para medir distancias de 2 cm a 400 cm con precisión de 3 mm. Emite un pulso ultrasónico por el pin Trig y mide el tiempo que tarda el eco en regresar por el pin Echo. La distancia se calcula con la fórmula: distancia = (tiempo × 0.0343) / 2.

✅ Ventajas
  • Muy económico (~$1-2 USD)
  • Fácil de usar con pulseIn()
  • No requiere contacto físico con el objeto
  • Funciona en oscuridad total
⚠️ Limitaciones
  • Superficies blandas o en ángulo pueden no reflejar bien
  • Ángulo estrecho: solo detecta lo que está al frente
  • No funciona bien bajo el agua o en polvo
hc_sr04.inoULTRASONIDO
const int TRIG = 9, ECHO = 10;

void setup() {
  pinMode(TRIG, OUTPUT);
  pinMode(ECHO, INPUT);
  Serial.begin(9600);
}
void loop() {
  digitalWrite(TRIG, LOW);  delayMicroseconds(2);
  digitalWrite(TRIG, HIGH); delayMicroseconds(10);
  digitalWrite(TRIG, LOW);

  long duracion = pulseIn(ECHO, HIGH);
  float dist = (duracion * 0.0343) / 2.0;
  Serial.print("Distancia: "); Serial.print(dist); Serial.println(" cm");
  delay(200);
}

PIR HC-SR501 — Sensor de Movimiento Infrarrojo Pasivo
🔍 Sensor de Presencia

El sensor PIR (Passive Infrared) detecta el calor infrarrojo emitido por personas y animales en movimiento. Tiene dos potenciómetros ajustables: sensibilidad (hasta 7 metros) y tiempo de retardo (2–300 segundos). Cuando detecta movimiento, su salida va a HIGH durante el tiempo configurado. Es el sensor estándar de alarmas y luces automáticas.

Servo Motor SG90 — Control de Posición Angular
🔄 Actuador Rotacional

El servo SG90 es un motor con engranajes y circuito de control integrado que permite posicionarlo con precisión en ángulos de 0° a 180°. Se controla con señales PWM: el ancho del pulso determina la posición. Incluye tres cables: rojo (5V), marrón (GND) y naranja (señal).

servo_sg90.inoSERVO
#include <Servo.h>
Servo miServo;

void setup() {
  miServo.attach(9); // servo en pin 9
}
void loop() {
  for (int pos=0; pos<=180; pos++) {
    miServo.write(pos);  // ir de 0° a 180°
    delay(15);
  }
  for (int pos=180; pos>=0; pos--) {
    miServo.write(pos);  // regresar a 0°
    delay(15);
  }
}

Módulo Relé — Control de Alta Potencia
🔌 Interruptor Eléctrico Controlado

Un relé es un interruptor electromecánico que permite a Arduino (5V, señales débiles) controlar circuitos de alta tensión (110V-220V AC, cargas de hasta 10A). Al aplicar voltaje a su bobina, cierra o abre contactos que controlan el circuito principal. Esencial para controlar focos, electrodomésticos o motores industriales.

Carga AC
250V / 10A
Carga DC
30V / 10A
Voltaje bobina
5V DC
Corriente bobina
~70-90 mA
Aislamiento
Galvánico
Tiempo conmutación
~10 ms
⚠️
¡Precaución con voltajes AC! Al trabajar con 110V o 220V, el riesgo de electrocución es real. Siempre usa el relé con la corriente desconectada al hacer conexiones. Nunca toques los terminales AC cuando el circuito esté energizado. Los módulos de relé para Arduino incluyen aislamiento óptico (optoacoplador) para proteger la placa.

HC-05 — Módulo Bluetooth
📱 Comunicación Inalámbrica de Corto Alcance

El HC-05 permite conectar Arduino con teléfonos Android, computadoras y otros dispositivos Bluetooth. Trabaja en modo maestro/esclavo y se comunica con Arduino mediante el protocolo UART (Serial). Con aplicaciones como "Serial Bluetooth Terminal" en Android se puede enviar y recibir datos de forma inalámbrica sin Wi-Fi ni internet.

¡Atención con el voltaje! El pin RX del HC-05 opera a 3.3V lógico. Si Arduino envía 5V al RX del HC-05, puede dañarlo. Usa un divisor de voltaje (2 resistencias) o un convertidor de nivel lógico entre Arduino TX (5V) y HC-05 RX.

LCD 16×2 con módulo I2C
📺 Display de Cristal Líquido

El LCD 16×2 muestra 2 líneas de 16 caracteres ASCII cada una. El módulo I2C (PCF8574) adaptador reduce la conexión de 16 cables a solo 4 (VCC, GND, SDA, SCL), liberando pines del Arduino. Soporta texto, caracteres personalizados y mensajes dinámicos. Es el display más económico y didáctico para proyectos de Arduino.

Módulo Tarjeta SD + RTC DS3231
📦 Almacenamiento y Registro de Datos

El módulo SD permite leer y escribir archivos en tarjetas MicroSD (FAT16/FAT32) usando el protocolo SPI. El DS3231 es un reloj de tiempo real (RTC) de alta precisión con memoria de batería que mantiene la hora exacta incluso sin energía. Combinados, forman un sistema de datalogger para registrar lecturas de sensores con marca de tiempo.

Componentes de Protección
Resistencias, diodos, capacitores y fusibles para proteger el circuito

En cualquier proyecto con Arduino, los componentes de protección son tan importantes como los funcionales. Sin ellos, un pico de voltaje, una polaridad invertida o una sobrecorriente pueden destruir permanentemente el microcontrolador o los componentes conectados. Cada componente tiene un rol específico en el circuito.

💡
Regla de oro: Siempre coloca una resistencia en serie con un LED conectado a Arduino. Calcula el valor con: R = (Vcc - Vled) / Iled. Para un LED rojo con Vled=2V, Iled=20mA en 5V: R = (5-2)/0.02 = 150Ω. Usa 220Ω como valor seguro universal para LEDs de 5V.

🎓 ¿Qué aprendiste?

Recorriste los 10 componentes y sistemas fundamentales de Arduino: desde entender qué es la plataforma y su filosofía open-source, hasta el corazón del sistema (ATmega328P), los pines de I/O digitales y analógicos, la alimentación, sensores de temperatura, distancia y movimiento, actuadores como servos y relés, módulos de comunicación Bluetooth y Wi-Fi, pantallas LCD, almacenamiento en SD y los componentes de protección que mantienen todo funcionando.
